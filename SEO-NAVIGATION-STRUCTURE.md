## Navigation State Management

```
Desktop Megamenu State:
┌─────────────────────┐
│ isOpen: boolean     │ ← Menu visible/hidden
│ buttonRef: RefObject│ ← Reference to trigger button
│ menuRef: RefObject  │ ← Reference to dropdown
└─────────────────────┘
         │
         ├──> Click button: Toggle isOpen
         ├──> Click outside: Close menu
         └──> Click link: Close menu + navigate

Mobile Accordion State:
┌─────────────────────┐
│ isOpen: boolean     │ ← Menu expanded/collapsed
│ activeColumn: number│ ← Which pillar is expanded
└─────────────────────┘
         │
         ├──> Click "Guides": Toggle isOpen
         ├──> Click pillar: Set activeColumn
         └──> Click link: Close all + navigate
```

## Data Flow

```
1. Navigation Data Source (seo-megamenu.tsx)
   └─> SEO_NAVIGATION constant
       └─> Exported to other components

2. Header Component (header.tsx)
   └─> Imports and renders SeoMegaMenu
       ├─> Desktop: <SeoMegaMenu />
       └─> Mobile: <SeoMegaMenu isMobile />

3. SEO Index Page (app/[locale]/seo/page.tsx)
   └─> Imports SEO_NAVIGATION
       └─> Renders guide cards with links

4. Sitemap (lib/sitemap-seo.ts)
   └─> SEO_PAGES constant
       └─> generateSeoSitemapEntries()
           └─> Used in app/sitemap.ts
```

## Performance Optimization Strategy

```
1. Static Generation
   └─> All 50 pages pre-generated at build time
       └─> Instant page loads

2. Code Splitting
   └─> Megamenu loaded only when opened
       └─> Reduced initial bundle size

3. Lazy Loading
   └─> Images below the fold loaded on scroll
       └─> Faster initial paint

4. Caching
   └─> Static pages cached by CDN
       └─> Sub-second global response times
```
